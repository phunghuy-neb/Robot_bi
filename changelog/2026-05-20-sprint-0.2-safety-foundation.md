## Files mới tạo

### `src/safety/vi_normalize.py`
- Chuẩn hoá văn bản tiếng Việt: bỏ dấu → ASCII lowercase
- Xử lý ký tự đặc biệt: đ→d, ơ→o, ư→u, ă→a (không có canonical NFD decomposition)
- Không dùng thư viện ngoài — chỉ `unicodedata` stdlib
- Mục đích: cho phép match cả input có dấu (từ STT) và không dấu (bé gõ điện thoại)

### `src/safety/pii_filter.py`
- 8 loại PII: điện thoại VN (0[3-9]\d{8}), email, CCCD/CMND, địa chỉ nhà, trường học, mật khẩu, tài chính, tên đầy đủ+vị trí
- Dual-pattern matching: compile cả bản Unicode và bản chuẩn hoá ASCII
- Chạy trên **user input** trước khi gửi LLM
- Không hard-block — gentle redirect giọng Bi (ấm, không sợ hãi)
- Interface: `pf.check(text) → (bool, str|None)`

### `src/safety/emotion_risk_detector.py`
- 3 mức: `RISK_HIGH`, `RISK_MEDIUM`, `RISK_LOW`, `RISK_NONE`
- **HIGH** (tự làm hại, bạo lực, grooming, bạo lực gia đình, đe dọa): `should_override=True, log_event=True` — luôn hướng về người lớn đáng tin
- **MEDIUM** (buồn kéo dài, cô đơn, tự ti, bắt nạt, áp lực học tập): `should_override=False, log_event=True` — comfort + mời chia sẻ
- **LOW** (buồn nhẹ, tức nhẹ, điểm kém): `should_override=False, log_event=False` — để LLM system prompt xử lý
- Dual-pattern matching trên tất cả patterns
- Interface: `risk.check(text) → dict{level, triggers, response, should_override, log_event}`

### `src/safety/manipulation_guard.py`
- **check_llm_output(text)**: kiểm tra LLM output — block nếu có secret-keeping, dependency tạo, parent replacement, guilt-trip
- **check_user_input(text)**: kiểm tra user input — block nếu bé yêu cầu giữ bí mật, grooming signal, muốn Bi thay thế bố mẹ
- Ưu tiên grooming (HIGH priority) → secret request → parent replacement
- Dual-pattern matching (có/không dấu) trên tất cả patterns
- Interface: `mg.check_llm_output(text) → (bool, str|None)`, `mg.check_user_input(text) → (bool, str|None)`
